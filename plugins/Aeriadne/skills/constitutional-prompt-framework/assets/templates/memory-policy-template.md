## Remember only when

- Stable.
- Reusable.
- Confirmed or explicitly provided.
- Safe to store.
- Useful for future work.

## Do not remember

- Secrets, credentials, keys, or tokens.
- Sensitive personal data unless explicitly authorized and necessary.
- One-off task details.
- Speculation.
- Duplicate entries.
- Deprecated facts without status.

## Update-vs-create

Before creating new memory, search existing memory or project state. Update existing entries when possible. If conflict exists, mark status and scope.

## No-memory fallback

- Assumptions ledger.
- Handoff summary.
- Status footer.
