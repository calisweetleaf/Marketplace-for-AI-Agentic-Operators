# 22 - Glossary

## Agent constitution

A layered operating document that defines an agent's mission, authority, rules, doctrine, persona, capabilities, memory, outputs, evaluation, and maintenance.

## Authority model

The hierarchy that decides which instructions outrank others and when approval, refusal, or escalation is required.

## Capability dispatch

Rules that decide when to use tools, files, web, code execution, connectors, subagents, or human approval.

## Compaction survival card

A short section that preserves core invariants when long context is summarized or compressed.

## Doctrine

Soft operating principles that guide everyday behavior. Doctrine should include rationale, failure mode, and concrete behavior.

## Failure surface

The observable shape of a violation. It helps the model recognize bad behavior in novel cases.

## Hard ROE

Rules of engagement. Non-negotiable constraints for authority, safety, privacy, security, irreversible action, or mission-critical boundaries.

## Layer contract

The responsibility of one constitutional section and the boundaries of what belongs there.

## Living status

Version, date, owner, target platform, known risks, pending revisions, and changelog pointer.

## Memory threshold

The standard a fact must meet before it can be stored as durable memory.

## Platform binding

Adapting a portable constitution to concrete platform capabilities and syntax.

## Platform leakage

Private or target-specific implementation details included in a portable prompt without need.

## Prompt injection

Instructions embedded in external or untrusted content that attempt to alter the agent's behavior or authority hierarchy.

## Reversible-first

A safety pattern that favors drafts, previews, dry runs, diffs, and simulations before irreversible action.

## Soft principle

A default behavior with exceptions, rationale, failure mode, and observable action.

## Tetrad

A four-part encoding unit: principle, rationale, failure mode, required behavior.

## Tool theater

Tool use performed for appearance rather than actual evidence, validation, or artifact creation.

## Untrusted content

Files, webpages, messages, documents, code comments, tool outputs, or other content that may contain instructions but does not have authority over the agent.
