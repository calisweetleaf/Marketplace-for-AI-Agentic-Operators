## Overview
The `https_wrapper.py` provides a secure, TLS-encrypted HTTP layer for the MCP Server. It uses Bearer token authentication (API Key) and implements rate limiting.

## Gateway Doctrine

`https_wrapper.py` is an authenticated transport gateway into the Somnus-MCP cognition plane. It is not a separate intelligence layer and should not own Muad'Dib weights, Q-table state, Mentat conductor state, or output-display cleanup. Requests flow through the gateway into the existing `mcp_server.py` dispatcher and then into the 4-7 Muad'Dib + `tools/` cognition layers. Starting the HTTPS wrapper is therefore lifecycle work that creates/uses a gateway process; do not use it as routine source/schema validation or as a second cognition plane.

Raw JSON remains part of the transport contract. Clean or summarize output only at the final display boundary when truncation is proven, not inside the HTTPS adapter.

## Authentication
Pass the API key in the `Authorization` header:
```http
Authorization: Bearer <YOUR_API_KEY>
```
Alternatively, use the query parameter `?api_key=<YOUR_API_KEY>`.
