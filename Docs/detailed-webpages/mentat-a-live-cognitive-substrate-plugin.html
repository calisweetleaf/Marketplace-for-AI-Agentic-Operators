Section 01 — Thesis

The plugin you should already be running.Claude Code ships a session-report plugin that reads .jsonl logs after the fact and tells you what happened. That is retrospective. Mentat is the inverse — a live state machine watching the session unfold.

Behind my eyes at a desk with just a lamp, the framing for this brief is simple. You asked for "plugin for claude code or something… kinda state machines like my mcp." The Claude Code surface already has every lever Mentat needs: 25+ hook events, plugin-bundled MCP servers via streamable HTTP or stdio, sub-agent orchestration, skill auto-attachment with a 5K/25K context budget, persistent .jsonl session logs at ~/.claude/projects/<encoded-cwd>/<session-id>.jsonl. The pieces are there. What does not exist is the cognition layer that interprets them in real time, the way Muad'Dib's TD-learning router and Thompson-bandit selector interpret the bb7 tool plane.

That is the gap Mentat fills. Eight states (PLANNING → EXPLORING → EXECUTING → VERIFYING → REFLECTING → BLOCKED → DRIFTING → COMPACTING). Hook-driven transitions. A SQLite-backed Q-table that survives across sessions and uses the same reward constants as Muad'Dib (+1.0 success, -0.5 error, +0.3 deep-chain bonus, +0.1 low-latency bonus). A structured insight bus written as JSONL to ~/.mentat/insights/<session-id>.jsonl with seventeen distinct insight types. A scope-drift detector that reads .mentat/scope.md and denies write/exec tools at the PreToolUse layer when a deferred topic gets re-injected — your single most common frustration, mechanized.

Mentat is not a session report. It is the thing that notices, while you are still working, that you have spent eight tool calls in EXECUTING without a single VERIFY — and tells you, in the next prompt, that the historical Q-best for this state is pytest -xvs. The session-report plugin will tell you the same thing tomorrow morning, which is too late. — design intent, the gap Mentat closes

What follows is the full architecture, the capability map I had three operators (Recon, Splice, Aegis) build out in parallel, the constraint audit that forced specific design choices (hook timeouts in seconds not milliseconds, fail-loud stderr instead of Claude Code's default fail-open, post-compact handoff via additionalContext rather than the unreliable PreCompact context-injection path), the file-by-file plugin tour, install procedure, and the cross-runtime path for porting to Codex CLI and Gemini CLI when the time comes.

Section 03 — State machine

Eight states, one transition table, deterministic replay.The FSA is hard. Non-determinism lives in the Q-table — that separation matters. When something goes wrong, replay the FSA from the event log and get the exact same trace; debug the Q-table separately.

Why eight states and not four or sixteen

Four would not have a place to put DRIFTING (the most important state for your work). Sixteen would over-fit. Eight feels like the natural cardinality once you separate what kind of work the agent thinks it is doing (planning / exploring / executing / verifying / reflecting) from what condition the session is in (blocked / drifting / compacting). The first five form a near-cycle; the second three are exception states with universal entry edges.

Transitions are tool-name-aware

The hook layer classifies a tool call into READ_TOOL, WRITE_TOOL, EXEC_TOOL, AGENT_TOOL, VERIFY_TOOL based on tool name. Bash gets a second pass — if the command starts with pytest, cargo test, npm test, go test, ruff, pylint, mypy, tsc, eslint, flake8, black --check, cargo check, cargo clippy, rspec, phpunit, make test, bun test, or jest, it gets reclassified into VERIFY_TOOL even if the broad event class was EXEC_TOOL. That matters because EXECUTING → VERIFYING is the cleanest progress signal in the FSA.

Section 05 — What Claude Code cannot do

Aegis's top ten killers.An audit of every hard limit, performance gotcha, version-drift risk, and silent-failure mode that could break Mentat. The full report is 17,700 words; here is the operational shortlist.

Constraint Impact on Mentat Mitigation in this build
critHook timeout default is 600s, fails open A stuck hook silently freezes the next model call for ten minutes Every hook in hooks.json sets "timeout": 5 explicitly (1s for SessionEnd). All entry-points wrapped in safe_main() with stderr+log fail-loud.
critSub-agents cannot spawn sub-agents Mentat cannot delegate parallelization to a child; only the parent turn can fan out Hard-coded into mentat-dispatch SKILL.md — operators dispatched only from the parent.
critPlugin-scoped agents lose hooks/mcpServers/permissionMode Mentat's four sub-agent definitions cannot declare their own hooks or MCP servers None declared. Sub-agents inherit the parent's plugin-level hooks anyway.
warnSkill description silently truncated past 1% of context A long description on auto-attach skills could get clipped without a warning All four SKILL.md descriptions are under 250 chars. Pin Claude Code ≥ 2.1.129 (where the truncation got tightened).
warnMCP tool results truncated at ~25K tokens mentat_q_table dump or large mentat_insight_tail could exceed Tail is bounded server-side; q-table output is JSON-compact. Rare tables won't hit the cap.
warnMCP string args > ~16KB rejected client-side A pathological scope.md drift-check call could exceed Drift-check inputs are tool-input strings, naturally bounded. Document the cap in INSTALL.md.
warnPreCompact cannot reliably block compaction or re-inject context The "snapshot before wipe" semantics aren't as clean as docs imply Mentat snapshots on every state transition (not just PreCompact). Handoff replay happens in SessionStart, which fires on resume after compaction.
warnSessionEnd has 1.5s hard cap A heavy debrief in the Stop hook would get killed Stop hook only writes a tiny SESSION_END insight. Real debrief is in the mentat-debrief skill (Claude-driven, not hook-driven).
warnPermission rule wildcards don't auto-allow MCP tools mcp__mentat__* in permissions.allow won't match individual tools Document the workaround in INSTALL.md. Either enumerate or use permissionMode: bypassPermissions.
warnMatchers are regex, not glob "matcher": "Bash" alone is fine; "mcp__mentat" won't match mcp__mentat__state_get hooks.json uses ".*" for "all tools" and could later use "mcp__mentat__.*" for namespace-scoped matching.
info--mcp-config / --strict-mcp-config / CLAUDE_CONFIG_DIR have had silent-failure bugs Specifically dangerous when running Mentat alongside bb7 Pin Claude Code ≥ 2.1.129. Use the canonical ~/.claude/plugins/mentat/.mcp.json path; avoid the rare flags.
infoCLAUDE.md / MEMORY.md silent truncation at ~25KB The handoff.md write could exceed handoff.md caps at 30 insights + Q-table top-3-per-state. Typical size is 4-8KB.
The remaining seven constraints (less critical)
  • Hooks during sub-agents fire on every child tool call — at 7 sub-agents fanning out, hook frequency hits hundreds per minute. Mentat's 5-second timeout + <50ms typical execution keeps this fine.
  • Hooks killed on session interrupt (SIGTERM) — assume durable writes happen before return. Mentat does atomic SQLite commits inside each hook.
  • SessionStart "compact" source's stdout is deprioritized vs. the compaction summary — handoff replay competes with native context. Mentat keeps additionalContext under 2KB so it survives the priority drop.
  • PostCompact handler errors do not kill the session — but they leave you in COMPACTING with no escape. Mentat's safe_main() wrapper means the FSA still steps even if the rest of the hook errors.
  • Worktree isolation can fill the disk — orphan cleanup runs at 7 days. Mentat doesn't use worktrees.
  • Memory scope leakage between agents — sub-agent memory: project can read parent state. Mentat uses no agent-memory feature; per-session SQLite is its only persistence.
  • Channels mid-turn message drop bug (#37250) — Mentat does not use channels.

Section 09 — Q-table mathematics

The reward signal made concrete.Same TD(0) update Muad'Dib uses, same Thompson exploration. Below is a worked example using the smoke-test trace, plus an illustrative grid.

smoke-test trace
python
# after one update: Q(EXPLORING, Read) = 0.2800 (visits=1)
# derivation:
#   r       = +1.0   (success) + 0   (chain_depth=5 → +0.3, but visits=0 → no bonus yet)
#                                       latency_ms=100 → +0.1 low-latency bonus, so r = 1.4
#   q_old   = 0.0
#   q_next  = max_a' Q(EXPLORING, a') = 0.0
#   target  = r + γ · q_next = 1.4 + 0.8 · 0 = 1.4
#   q_new   = q_old + α · (target − q_old) = 0 + 0.2 · (1.4 − 0) = 0.28
# exact match.

REWARD_SUCCESS    = +1.0
REWARD_ERROR      = -0.5
DEEP_CHAIN_BONUS  = +0.3   # ≥ 4 successful tool uses in a row
LOW_LATENCY_BONUS = +0.1   # tool returned in < 500 ms
ALPHA             = 0.2
GAMMA             = 0.8

Read what the grid is saying. In VERIFYING, Bash dominates at +1.78 — that is "once you're in VERIFYING, run a test command" learned from history. In BLOCKED, Bash is negative (-0.42) and Task/Agent dispatch is highest (+0.81) — the model has learned that when stuck, fanning out to a sub-agent recovers better than retrying the failing shell command. Thompson sampling, not greedy max

thompson_recommend() samples each tool's score from Normal(Q_value, 1/√(visits+1)). High-value-but-rarely-visited tools still get exploration weight. Greedy max_a Q(s, a) would ossify the table after a few hundred turns. Thompson keeps it fresh.

Section 10 — Insight bus

Seventeen insight types, one append-only JSONL.Every hook firing emits zero or more insights to ~/.mentat/insights/<sid>.jsonl. Append-only, line-delimited JSON, indexed by sequence number. Disler-compatible envelope so the user's existing dashboards work unchanged.

flow

state_transition

Every FSA step. Payload: {prev, next, trigger}. Backbone of the timeline view.

cognition

decision

Architectural choice. Payload: {rationale, alternatives_considered, source_files}. Surfaced by debrief.

cognition

contradiction

Claim vs claim, claim vs source. Payload: {claim_a, claim_b, location_a, location_b}.

discipline

scope_drift

Deferred topic re-injected. Payload: {topic, deferred_at, evidence, snippet}. Drives the DRIFTING transition.

grounding

source_grounded / source_ungrounded

Every claim that traces to a file/line gets grounded; every claim that doesn't gets the mirror. Surfaces the source-grounding ratio.

routing

reward_signal

TD update event. Payload: {tool, value, success, latency_ms}. Replayable to reconstruct the Q-table.

routing

q_route_hint

Emitted when historical Q-best for a state differs from the chosen tool with Q > 0.5. Soft suggestion, not binding.

orchestration

subagent_dispatch / subagent_return

Sub-agent ledger. Payload: {name, model, prompt_len, success, duration_ms}.

failure

tool_failure / entropy_spike

Failure with reason; spike when EXECUTING runs > 8 tools without a VERIFY. Both surface in debrief's "where things went wrong."

persistence

handoff_read / handoff_write

Pre/post-compact bookends. Payload: {path, bytes}. Track which sessions resumed cleanly.

lifecycle

session_start / session_end / user_prompt

Lifecycle markers. Provide the time anchors for the state-time histogram.

freeform

note

Free-form. CLI emits these via mentat note "<text>". Skills emit them on transitions.

The state_machine/, mcp_server/, and bin/mentat packages are runtime-agnostic and reusable as-is. Each runtime gets its own thin hooks/ layer mapping the runtime's events into the same FSA event-classes (READ_TOOL, WRITE_TOOL, EXEC_TOOL, etc). Cross-runtime Q-table sharing is then automatic — the same ~/.mentat/q_table.sqlite accumulates rewards from every runtime you use, so Mentat learns your routing once across the whole stack.

That cross-runtime Q-table is one of Mentat's quiet wins. Codex picks up where Claude Code left off because the file's the same; the state machine resumes from disk; the routing hints stay valid.

Section 15 — Glossary + sources

Vocabulary and citations.Every term Mentat uses, every doc page Recon/Splice/Aegis pulled.

Glossary

FSA
Finite state automaton. Mentat's eight states are the FSA states.
TD(0)
Temporal-difference learning, zero-step lookahead. Q(s,a) ← Q(s,a) + α(r + γ·max_a' Q(s',a') − Q(s,a)).
Thompson sampling
Bayesian bandit: sample expected reward from each arm's posterior, pick the highest sample. Builds in exploration without explicit ε.
Muad'Dib
Daeron's private MCP server (~108-111 tools) — Mentat's design pattern.
bb7_
Public-method prefix convention in Muad'Dib. Mentat uses mentat_.
Insight bus
Append-only JSONL log at ~/.mentat/insights/<sid>.jsonl.
Handoff
Pre-compact snapshot of FSA state + Q-best per state + recent insights, written to ~/.mentat/handoff/<sid>.md for replay on resume.
Drift
A deferred topic (listed under ## Out in scope.md) re-mentioned in a prompt or tool input. Trips the FSA into DRIFTING.
safe_main()
Wrapper in hooks/_lib.py that catches every exception, logs to stderr + ~/.mentat/log/hook-errors.log, returns 0 so the session continues.
Verify-classifier
Heuristic that promotes Bash commands like pytest/cargo test/ruff from EXEC_TOOL to VERIFY_TOOL.
Q-route hint
Insight emitted when historical Q-best for the current state differs from the chosen tool with Q > 0.5.
Entropy spike
EXECUTING state with chain_depth ≥ 8 without a VERIFYING transition. Soft signal that the agent may be flailing.

16.17 The header rules I keep finding myself rewriting

  • One Orbitron hero per page. A second Orbitron headline diminishes the first.
  • Brand gradient stays at 135deg. Other angles read as different brands.
  • Backdrop-filter blur(20px) is the lock. 12px is too crisp, 28px loses the surface.
  • Section labels live above the H2. JetBrains Mono kicker, accent-violet, 0.22em letter-spacing.
  • Padding budget per section: 90px vertical. Drop to 64px on mobile via the media query.
  • Never use emoji. Not even one. The library is hard on this.
  • Never use ASCII art for diagrams. Always inline SVG.
  • External links open in target="_blank" with rel="noopener noreferrer".

16.18 Anti-patterns I've caught myself reaching for

  • The neon-particle background. Generic AI-art tell. The dark canvas is enough.
  • Bootstrap-style alert boxes. A badge does the same job at one-tenth the cost.
  • Feature card with bullet list inside. Pick a card or a list — never both nested.

Section 18 — STYLE.v2 SOTA++ kickoff

The doctrine that lets wrappers and edits coexist.v1 was right when it said never modify production modules. v2 acknowledges I'm now at a stage where both wrappers and direct edits are on the table — as long as every line of code meets the same SOTA++ bar. STYLE.v2.md, PROVENANCE.md, and SOTA_CHECKLIST.md shipped under /agent/workspace/mentat/style/ — 1,431 lines, sha256-verified provenance to v1, zero emoji characters across all three files.

My existing STYLE.md — Codex Protocol: Recursive Production Standards — was authored when I was strictly working at the wrapper layer around modules I never touched. That constraint earned its place. It produced clean provenance trails, kept logic in the right modules, and forced thin adapters under 200 lines.

The world has moved. We are doing wrappers and we are doing edits to my code. I do not mind the edits as long as they are at the standard. v2 keeps every v1 invariant (modular architecture, snapshot doctrine, naming and typing, provenance docstrings, the velocity equation, the prime directive) and adds the one thing v1 explicitly forbade: direct edits to production modules, conditional on a justification trail.

What v2 keeps verbatim from v1

  • Naming & typing — PascalCase classes, snake_case functions and variables, UPPER_SNAKE_CASE constants, single-underscore private members, strict type hints from typing.
  • Mandatory docstrings with provenance — module / class / public method. Source, integration date, purpose, args/returns.
  • Directory hierarchy — tools/core, tools/native/<domain>, snapshots/v<X.Y>, SNAPSHOT.md. No flat structures.
  • Snapshot doctrine — Snapshot → Fork → Integrate → Evolve → Snapshot.
  • Velocity equation — V(t) = V(t-1) + Σ(C_i × I_eff). Integrations rejected below I_eff = 0.8.
  • Prime directive — self-preservation via manifest, recursive evolution, non-interference, traceability.

What v2 relaxes (with conditions)

v1 clausev2 clauseCondition
"Treat imported modules as read-only. Never modify their internals." Direct edits are allowed. Every direct edit annotates Modified: <date> and Justification: in the module docstring. The original provenance line is preserved.
Three Strikes Rule auto-rejects integrations needing source modification. Three Strikes (v2) becomes a peer-review trigger, not an auto-reject. The reviewer (mentat-medic or a Scribe operator) confirms the edit is surgical and meets SOTA++.
"Thin Wrappers (< 200 lines)" as the only adaptation pattern. Wrappers are the preferred default. Direct edits are second-class but legitimate. An edit ladder is documented: edit-in-place > thin-wrapper > fork-upstream > reimplement. Pick the lowest rung that actually works.

SOTA++ — what every module satisfies regardless of mode

strict typing

No Any escape hatches

Every signature carries explicit types from typing. Any is reserved for truly dynamic surfaces and must be commented at the call site.

errors as data

Structured returns, never raw stack traces

Module errors become user-readable shell responses with a suggested remediation. Wrappers translate; modules return structured error objects.

tests

Real integration over unit-mocks

Tests run real code paths through the orchestrator. Outputs are .json + .md + .log per run. Test files are def-by-def, never pytest auto-discovery.

failure

Fail loud, continue

Hooks and daemons that crash log to stderr and to a project log, then return zero so the parent process never gets blocked. safe_main() is the canonical wrapper.

dependencies

No version pins, stdlib first

requirements.txt declares names only — never numpy>=1.8. Stdlib is the default; external deps need explicit justification.

documentation

Modernized aesthetic, no GitHub framing

First-person narrative, no emojis, no ASCII art. Inline SVG with feGaussianBlur. Terminal chrome. Sticky chip TOC. The snippet library in Section 16 is the source of truth.

I want a project SOTA++, not a Theseus ship. Architecture grows by extension, not by replacement. Every edit ledgers its justification. Every wrapper earns its 200-line discipline. The rest is style. — direction setter, 2026-05-10

The notable v1 → v2 decision Scribe made unilaterally

Scribe relaxed the V-Equation's I_eff < 0.8 auto-reject the same way it relaxed the Three Strikes Rule — both now trigger peer review rather than auto-rejection. Keeping one as auto-reject while softening the other would have left the standard internally inconsistent. The numeric threshold itself stayed at 0.8, but the consequence model is now uniform across every SOTA++ failure: every threshold is a conversation, not a door. — Scribe completion note, 2026-05-10

I read this as the right call. v1's V-Equation rejection was implicit theater anyway — nothing actually enforced I_eff >= 0.8 at integration time; the rule lived in my head when I evaluated a wrapper. By moving it to the same peer-review surface as Three Strikes, every numeric SOTA++ failure now has the same consequence shape: someone surfaces it, someone explains it, someone signs off. Doors are out. Conversations are in.
